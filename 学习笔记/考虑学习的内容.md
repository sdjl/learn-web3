# 考虑学习的内容

## 说明

**只学习需要开发者主动编写代码的功能。**

工具库自动完成的功能（如转账时的自动签名、钱包连接时的自动处理等）不需要学习。这里列出的都是需要开发者使用 Hook 或 API 主动实现的功能。

## 推荐学习内容

### 合约交互

#### 3. 读取合约状态

- **学习目标**: 使用 `useReadContract` Hook 读取智能合约数据
- **应用场景**: 查询合约状态、读取公开变量
- **相关 Hook**: `useReadContract`, `useReadContracts`
- **难度**: ⭐⭐⭐
- **建议**: 可以先部署一个简单的合约（如计数器合约）进行练习

#### 4. 写入合约

- **学习目标**: 使用 `useWriteContract` Hook 调用合约方法
- **应用场景**: 调用合约的写入方法、执行合约交易
- **相关 Hook**: `useWriteContract`, `useWaitForTransactionReceipt`
- **难度**: ⭐⭐⭐⭐

#### 5. 监听合约事件

- **学习目标**: 使用 `useWatchContractEvent` Hook 监听合约事件
- **应用场景**: 实时监听合约事件、更新 UI
- **相关 Hook**: `useWatchContractEvent`, `useWatchContractEvents`
- **难度**: ⭐⭐⭐

### 代币操作

#### 6. ERC-20 代币转账

- **学习目标**: 实现 ERC-20 代币的转账功能
- **应用场景**: 发送代币、代币支付
- **相关 Hook**: `useWriteContract` (调用 transfer 方法)
- **难度**: ⭐⭐⭐
- **建议**: 在 Sepolia 测试网上使用测试代币

#### 8. 代币授权（Approve）

- **学习目标**: 实现代币授权功能
- **应用场景**: DEX 交易前授权、DeFi 协议交互
- **相关 Hook**: `useWriteContract` (调用 approve 方法)
- **难度**: ⭐⭐⭐⭐

### 高级功能

#### 9. 类型化数据签名（EIP-712）

- **学习目标**: 使用 `useSignTypedData` Hook 实现结构化数据签名
- **应用场景**: 更安全的签名方式、DeFi 协议交互
- **相关 Hook**: `useSignTypedData`, `useVerifyTypedData`
- **难度**: ⭐⭐⭐⭐

#### 10. Gas 费用估算

- **学习目标**: 估算交易 Gas 费用
- **应用场景**: 显示预估费用、优化交易成本
- **相关 Hook**: `useEstimateGas`, `useGasPrice`
- **难度**: ⭐⭐⭐

#### 11. 批量读取合约

- **学习目标**: 使用 `useReadContracts` Hook 批量读取多个合约数据
- **应用场景**: 优化性能、减少请求次数
- **相关 Hook**: `useReadContracts`
- **难度**: ⭐⭐⭐

### NFT 相关

#### 13. 查询 NFT 余额

- **学习目标**: 查询地址拥有的 NFT 数量
- **应用场景**: NFT 展示、NFT 钱包
- **相关 Hook**: `useReadContract` (ERC-721/ERC-1155)
- **难度**: ⭐⭐⭐

#### 14. 查询 NFT 元数据

- **学习目标**: 获取 NFT 的名称、图片、属性等元数据
- **应用场景**: NFT 展示、NFT 市场
- **相关工具**: IPFS、NFT API
- **难度**: ⭐⭐⭐⭐

## 学习建议

### 优先级排序

1. **高优先级**（基础必备）:

   - 切换网络
   - 读取合约状态
   - 写入合约

2. **中优先级**（常用功能）:

   - ERC-20 代币转账
   - 查询代币余额
   - 查询交易详情

3. **低优先级**（高级功能）:
   - 类型化数据签名
   - Gas 费用估算
   - 批量读取合约
   - NFT 相关功能

### 学习路径建议

1. **第一阶段**（巩固基础）:

   - 切换网络

2. **第二阶段**（合约交互）:

   - 读取合约状态
   - 写入合约
   - 监听合约事件

3. **第三阶段**（代币操作）:

   - ERC-20 代币转账
   - 查询代币余额
   - 代币授权

4. **第四阶段**（高级功能）:
   - 类型化数据签名
   - Gas 费用估算
   - 批量操作

### 实践建议

- 使用 **Sepolia 测试网**进行所有练习
- 可以部署简单的测试合约（如计数器、简单代币合约）
- 参考 [Wagmi 官方文档](https://wagmi.sh) 和 [Viem 文档](https://viem.sh)
- 每个功能实现后，按照规范编写学习笔记
